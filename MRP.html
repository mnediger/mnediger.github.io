POND Brain

Master's Research Project

Summary

This project addresses a communication gap between the scientists who study the neurobiological underpinnings of neurodevelopmental disorders and those who are personally affected by the disorders.

It's a web-based application that connects an interactive 3D brain model with descriptions of the structure and function of the brain.

As the sole designer and lead developer of the project, I was responsible for everything from defining the scope of the project to creating final project assets.

Project Demo

The goal of this project was to create an e-learning environment that demonstrates the structural and functional correlates of neurodevelopmental disorders. It was designed as an online resource for parents of children with neurodevelopmental disorders to learn about the neurobiological basis of these disorders.

It allows users to explore the structure of a typically developing brain by manipulating a 3-dimensional brain model and reading the associated plain language text. The text describes the notable functions of major functional areas, allowing users to click keywords in the text and reveal the corresponding components of the brain model. The application also features a novel method of visualizing neuroimaging data, in which the results of individual neuroimaging studies are displayed graphically on the brain model.

Process

Discover

Establish the problem

The Province of Ontario Neurodevelopmental Disorders Network Knowledge Translation team recognized that while parents of children with neurodevelopmental disorders express an interest in following research on neurodevelopmental disorders, but many are unable to use the scientific literature to do so.


Challenge

To improve the accessibility of research on neurodevelopmental disorders (ASD, OCD, ADHD) for a lay audience.


Define success

Through conversations with a number of project stakeholders and content experts, I established the 3 primary communication goals for the project. For the project to be considered successful, it had to address each of these problems, while meeting a few additional requirements.


Goals

  1. After using the application, users should be able recognize that different parts of the brain are responsible for different functions.
  2. The application should demonstrate that neurodevelopmental disorders are biological conditions, and that the behaviours associated with these conditions are not under voluntary control.
  3. After using the application, users should understand the major similarities and differences among neurodevelopmental disorders.

Requirements

The completed application must be:

  • Disseminable
  • Accessible
  • Modular
  • Flexible

Research

To design a tool aimed at helping others understand research on neurodevelopmental disorders, I needed to understand the research myself. I reviewed the literature and considered the background knowledge that would be necessary for a novice to grasp the information. I also spent time learning about the motivations, goals and needs of the potential users - eventually developing user personas and goal-based scenarios that helped me define important app functionality. Finally, I dug into the literature to discover best practices for communicating structure/function relationships, designing multimedia e-learning environments, and designing interactive learning experiences.


Design

Develop information architecture

Using my user personas and scenarios, I compiled a list of functional specifications around which to structure the project. Using these functional specifications and working alongside a content expect, I also created descriptions of the content that would occupy the application. From there I had everything I needed to sketch out a site map and establish navigation and content hierarchies.


Wireframe, prototype, iterate

Simple hand-sketched outlines gradually developed into comprehensive illustrations of site functionality. After developing each set of wireframes, I would return to project stakeholders for feedback before rethinking and refining my design. Following a few rounds of this process, wireframes evolved into low-fidelity, then high-fidelity prototypes.


Implement

Create 3D model

I extracted structural data from an MRI of a healthy, 6-year-old male brain to create rough 3D meshes of cortical and subcortical structures. These meshes were refined, retopologized, textured, and painted in ZBrush, then exported for use in the application. Each model underwent multiple rounds of optimization to be lightweight enough to be rendered in a browser.


Develop in HTML/CSS/Three.js

I divided the project into small, isolated prototypes to simplify development, eventually combining each prototypes to create a complete, functional application.
